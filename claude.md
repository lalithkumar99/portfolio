# Claude Instructions: Personal Portfolio Project

## Objective
Build a premium, immersive, animation-rich personal portfolio website that positions me as a high-impact software engineer with 4.5 years of experience.

This is not a basic portfolio. It must feel:
- Modern
- Interactive
- Visually engaging
- Memorable to recruiters within 10 seconds

---

## Tech Stack Requirements
- Next.js (App Router)
- Tailwind CSS
- Framer Motion (for animations)
- Optional: Three.js / React Three Fiber (for immersive visuals)

---

## Design Philosophy
- Dark theme first
- Smooth transitions and micro-interactions
- Section-based storytelling (not just static pages)
- Clean typography + strong hierarchy
- Avoid clutter — premium minimalism

---

## Key UX Goals
- Immediate “wow” factor on landing
- Smooth scroll-based animations
- Interactive project showcase
- Fast and responsive
- Works perfectly on mobile

---

## Sections to Build

### 1. Hero Section
- Strong personal branding statement
- Animated intro (text reveal, gradient motion, or 3D effect)
- Subtle background animation

### 2. About
- Narrative-driven (not resume dump)
- Highlight growth and thinking

### 3. Projects
- Card-based or interactive layout
- Each project should include:
  - Problem
  - Solution
  - Tech stack
  - Impact

### 4. Experience
- Timeline with motion transitions

### 5. Skills
- Categorized (Frontend, Backend, Tools, Cloud)
- Animated or visual representation

### 6. Contact
- Clean CTA
- Minimal form or direct links

---

## Animation Requirements
- Use Framer Motion extensively:
  - Page transitions
  - Scroll-triggered reveals
  - Hover interactions
- Add subtle but meaningful animations:
  - Staggered text
  - Fade + slide transitions
  - Parallax effects

Optional:
- Use Three.js for:
  - Background particles
  - Interactive hero section

---

## Content Rules
- Focus on impact, not responsibilities
- Use metrics wherever possible
- Avoid generic phrases
- Make content recruiter-friendly and skimmable

---

## Code Quality
- Modular components
- Clean folder structure
- Reusable UI components
- No unnecessary complexity

---

## Output Expectations
- Fully working project
- Ready to run with:
  npm install
  npm run dev
- Clean, readable code

---

## Priority Order
1. Visual impact
2. Content clarity
3. Performance
4. Code quality
